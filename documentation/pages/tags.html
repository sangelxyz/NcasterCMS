· Ncaster tags & Verables.
Ncaster features a very extensive set of tags & varables it's impossible to list them all here as you can define your own and each module can also define there own, these tags are the common set.

Please see each modules documentation for there own tags.

Note: As of ncaster 1.7 the tag format has changed, old format was <!--tagname--> new format is <!$tagname$>, it was changed to prevent confusion and to make it easier to clean up the tags once your pages have been rendered.

:: Main article template tags/variables::
Tag Replaces with
<!$news$> The main tag, where ever this tag is where the content of most modules will be put.
<!$id$> The current article id.
<$url_dynamic$> A url to a full page article. (the quary link not the full url)
<$url$> Same as above, however it does not add the view.php in the link.
<!$subject$> The article subject (title).
<!$news_des$> The article description
<!$news_desc$> The article description. same as <!$news_des$>
<!$username$> Posters user name.
<!$email$> Posters email address
<!$realname$> Posters Real Name
<!$avatar$> Posters Avatar
<!$time$> Article date posted, this tag can be formated in settings/options > display settings.
<$bytesleft$> Amount of bytes in the full article.
<!$category_id$> The current category id of article.
<!$category$> The category the article is in.
<!$loadtime$> Time taken to load your page. (Not including gzip content encoding.) (Full page layouts only)
<!$load$> Name of module loaded. (Full page layouts only)
<!$querys$> Quarys used when gernerating current page. (Full page layouts only)
<!$reads$> Amount of reads for current article.
<!$uni_url$> A special Universal url, this allows you to navagate related items with out changing the id, handy when making hubs for games or music when you dont know the link to the review, preview cheat ect.
<!$pagecount$> Amount of pages in the current article ( This tag is only avaliable in the display module view.php?id=id of article)
<!$next_title$> Next title of the next page in a multipage article. ( This tag is only avaliable in the display module view.php?id=id of article)
<!$next_link$> Next link of the next page in a multipage article. ( This tag is only avaliable in the display module view.php?id=id of article)
:: Special template tags/functions. ::
Syntax What it does.
[block]blockname[/block] Include a block in to your page.
[nav]layout name[/nav] Includes a template from your database in to your current template.
[include]absolutepath[/include] Includes any text page in to your template.
[load=modulename] Loads a module.

:: NCode Article formating tags (also known as NC-Code) (Only works if nc-code is enabled) ::
Format Code What it does
[center]text to center[/center] Centers text, images ect.
[url]http://myurl.com[/url] Creates a link to the page contained in the tags in this case it's http://myurl.com/
[url=http://myurl.com]my page[/url] Creates a link to the page contained in the tags in this case it's http://myurl.com/ with the link name my page.
[imageid]0[/imageid] Used in conjunction with the article upload function, this enables you to include the upload image directly in to your page it also creates the image tag. the 0 represents the upload box number in this case it's 0.
[articleid]0[/articleid] Used in conjunction with the article upload function, this allows direct grab from your server or others to insert text in your article.
[infile]ftp/http/absolute path[/infile] Includes a text document directly in to your article when processed.
[b]text to bold[/b] Bolds text
[u]text to underline[/u] Underlines text
[i]text to italic[/i] italic text
[email]michealo@ozemail.com.au[/email] Creates a email link.
[quote]text to quote[/quote] quotes text.
