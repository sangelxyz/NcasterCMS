· Custom Fields
Over View: Ncaster is one of the very few that support the ability to add custom fields, you can also make your own field types and customize the way each field is displayed in your post screen each category can also be assigned it's own set of custom fields.

Tip: You can create your own drop down boxes, radio selects and you can then assign these to your custom fields.

- Creating/Editing/Deleting a custom field.
- Creating/Editing/Deleting a custom field type.
- Using Custom fields in your templates.

Creating & Editing custom fields.

First load your ncaster admin, click Settings > Add/Delete/Edit Fields.

1. Lession 1 Creating a field.

  • Click the Create button
  • You are now greated with a list of fields to fill out.
  • Field Name: This is the name of the field, it is also the name of the display tag for this field, do not
    include any special charactors in this field just a simple name is fine.
  • Field size & Field height: are how big the field will be in the post area.
  • Display Field: By selecting yes the field will be visable, by selecting no the field will not show in the post area.
  • Order Field: This is the order in which this field shows on screen.
  • Field Value: Allows you to set a value for the field, you can also use this in conjunction with your custom fields to select a value.
  • Once you have finished hit submit your field will then be saved.

2. Lession 2 Editing a field.

  • Click the Edit/Delete action button.
  • Click Edit on the field you wish to edit.
  • A list of fields are now displayed (same as above)
  • Once done changing click submit.

3. Lession 3 Deleting a field.

  • Click the Edit/Delete action button.
  • Click Delete on the field you wish to Delete.
  • The field is now removed.

1. You can include any number of <?php, <? start and end statments with in any template including news stlyes.
2. Php is very fast however it does not have the in built function handling of entity, if you make a function in a news style it will cause an error becouse it will be redecleared for each load, however if you put it in your full page template it will work fine.
3. Ncasters parser supports both <?php and <? php starting tags.
4. remember to return values that you want shown in the correct place.
5. You can access ncasters loaded classes directly such as Gcon (file opening functions, by remote or local), ncaster parser or the database connection engine, you can also load other classes or functions in to your scripts using phps include or require.
6. Ncaster has a built in libery of classes they range from zip, image manuplation (resize, watermark), file opening remote/local and article include.
7. Php is one of 2 scripting languages supported by the ncaster parser.
8. You can use php with entity and both can interact with each other.
