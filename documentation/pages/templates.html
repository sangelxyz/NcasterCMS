Templates Basic to Advanced Template Creation.

In this section we will cover.
* Creation of a basic template
* Apply advanced features and methods to our basic template.

Basic Template Creation.

Template types:
- Full page: a full page template is the actualy page it's self.
- News Style: a news style is a template that gets repeated over and over again to fourm your news page, this is then wraped in your full page template to give you your news page.

To create a new full page template in ncaster, click settings/options > Add Full page Layout
To create a new full page template in ncaster, click settings/options > Add News Display Template

now you are greated with the editor, the first thing you should do is give your template a name eg. My_full_page_template
and then a template.

A basic Full page template whould look something like this.
<html>
<head></head>
<title>My page: <!$category$>: <!$subject$></title>
<body>
<!$news$>
</body>
</html>

Note the comment tags <!$category$>, <!$subject$> & <!$news$> these among many others are parsed by ncaster before viewing, <!$news$> should be present in most if not all your templates with out it modules can not display there out put and you cannot view your news articles.

A basic News Style template whould look something like this.
<p><!$title$></b><br>
<!$news_desc$><br>
<a href="view.php<!$url$>">Full news</a> by <a href="mailto:<!$email$>"><!$username$></a></p>

After you have sucessfully created your templates you can assign each to categorys to be loaded you can do this by going to settings/options > template management, settings/options > Assign a template.

Template Naming system.
Syntex:
module:modulename{category:categoryid}{style:name of style};

examples:
Universal assign:
module:azlist;
Category Assign:
module:azlist{category:7};

Over view :
Ncaster features a template naming system that allows you to assign templates directly to modules, this naming system allows ncaster to be extreamly custimisable, each module can have

1 Universal template &
1 Category template

Each category in ncaster can also be given a Super Global template these templates are used for everything that uses that category however if you apply a universal template to a module it will be used for all categorys, if you apply a module-category template to that same module the category template will now be the display template.

Advanced Template Creation. When the basic stuff just ain't enought.
This is where the real fun starts the ncaster display engine is a multilayer rendering system that splits you template up when processed in to variables and multilayered html, variables and functions (entity) are rendered first this allows you to modifi tags values many times before the final tag parser is executed.

Lets take our basic news style and take and take a long look how we could improve it.
<p><!$title$></b><br>
<!$news_desc$><br>
<a href="view.php<!$url$>">Full news</a> by <a href="mailto:<!$email$>"><!$username$></a></p>

What i came up with.
- If the title is to long cut it down.
- if there is no news don't link.

lets apply these idears to our template using the new entity 2 scripting language.
<p>{$var|left|"$title;","16"}{$var|lengthstr|"$news_desc;"}{$var|morethen|"$var;","16",".."}</b><br>
<!$news_desc$><br>
{$var|lengthstr|"$news;"}{$var|morethen|"$var;","0","<a href="view.php<!$url$>">Full news</a>"} by <a href="mailto:<!$email$>"><!$username$></a></p>

this will cut the title string down to 16 bytes long and add the .. at the end if it's to big to fit.
the link will only be printed if <!$news$> field is more then 0 bytes long.

Nav includes (Template in template system): [nav]template name[/nav]
Nav includes works very similer to the php function include however the diffrence is that nav gets it's information from your database and not a file. The advantage of nav includes are that you can have one template that calls other templates to create a page e.g.

Include
Include Syntax: [include]absolutepath[/include]
Include is used to include external pages in to the ncaster display script you can include any file on the server except (config.php) by using the above syntax.

Entity: ncasters Native Template Scripting language

Require
Require Syntax: [require]absolutepath[/require]
In ncaster it is now possible to require in to your templates, this makes it possible to require polls and other programs directly in to your templates.
Note: Require only works in full page templates, Require is disabled by default to enable it open up view.php and edit the variable $requirefunction.

Otemp
Otemp is a feature of ncaster that allows you to overwrite an assigned template using a url you can change the template by doing something like this http://yourwebpage/view.php?otemp=templatename you can also use tid= which is similer to otemp however you call the id and not the name of the template e.g http://yourwebpage/view.php?tid=1 .
