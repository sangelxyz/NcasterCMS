· Ncaster Blocks System
Over View: Blocks are small files of code or html, ncaster can load an execute these files and integrate them in to your page.

Tip: If you create a block make sure you keep your code clean and try and reuse as much code as possible, Nuke Blocks can be integrated in to ncasters block system also however you will need to install nuke in most cases to use that feature.

Syntax .

[block]name of block[/block]

Using a block.

Blocks are supported in all full page templates and you can use more then one block to make up a page, blocks can contain any type of code however only php will be executed.

In one of your full page templates you can place the block include code any where with in it [block]block name[/block], the block will be executed and placed in the correct location of your page

Creating a block.

When creating Blocks you must follow the following rules.

  1. Make sure the file name ends with a .php extension and that this file is placed in the block folder.
  2. To stop the file from running as a standalone application insert the following bit of php in to your code
    if(preg_match("filename of your block",$PHP_SELF) ) { die('block cannot run in standalone mode') }

Developer notes .

  • When dealing with user input by Get or Post use the $input variable, this variable is populated by ncaster and is filtered.
  • You should try and limit the query's used by blocks if possible, ncasters debug functions should come in handy for working out the amount of query's your block is using to the amount of resources your block uses.
  • Use ncaster's functions where possible, for example a sql query use $sql->query("your query"); this allows for future compatibility with other databases.
  • If you have created a block please send it to michealo@ozemail.com.au these will be later included on the ncaster web site.
