· Associative articles
Over View: Associative articles is one of the many exclusive unique features you will find in ncaster 1.7, assiciative articles (also known as hubs) allow you to do the following.
* Assign a universal custom field group when edited these apply to all assiciated articles.
* articles relate, which opens up many more indexing methods such as showing the related articles.
* Allows you to bind the full data source to an article.
* Dramaticly decreases site maintence time.
* Non-Liner, create your own fields for the data sorce if needed, create more then one data souce(hub), each category can be given it's own data source(hub) which opens up easy management for even the biggest of sites.

Creating a hub.

Creating a hub is as simple as creating a category, just click Settings > Add a category
now select yes where it says, make this article an associative hub.
You can also change the hub back in to a category or change a category in to a hub using the edit category feature Settings > Edit a category.

Assiging a hub.

First make sure you create a hub next go to Settings > Edit a category, select a category you wish to assiciate this hub to, then select the hub you want to use from the drop down menu.

You can only assign one hub to each category however all categorys can have there own hubs.

Struturing your site to take advantage of AHubs.

Hubs save time by eleminating the need to copy data over and over though out articles, they also provide a quick and easy method of updating by just changing the hubs data all articles associated with it will see changers reflect imeditly.

In ncaster 1.6.5 and just about any premade news/cms or portal system you had the abilitys to create categorys these allow you to split up your data as much as possible with this type of system with ncaster 1.7 and the event of assiciative hubs and custom filtering it's possible eleminiate the need for some categorys.
