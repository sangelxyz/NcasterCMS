· Naming system
Over View: the template naming system was introduced to get at the hard to get sections in a module and allow you to skin them with html, the naming system is simple and in most cases you dont need to type a thing since a list of most templates names are generated for you in the edit/add template dialog.

How it works.

Ncaster uses a 5 tie template system it allows you to have One Global category template for each category, One Global module template for each module, One Global template for each modules category & One template per style defined in each module, this provides you will absolutly 100% control of the final out put if the module supports the naming system.

Naming.

Keys used in this syntex table.
%modulename% = the module you are assiging this template to.
%style% = the style you are assiging this template to, styles are defined in each module.
%category% = the category to assign this module to, must be of integer type. ex: 7

Title name assigns to:
module:%modulename%; %modulename% this template will be shown on any page using this module.
module:%modulename%{category:%category%}; the specified %category% in %modulename%, it will be shown on any page in that module using that category.
module:%modulename%{category:%category%}{%style%}; the specified %style% in %category% & %modulename%, it will be shown on any page in that module using that category.
module:%modulename%{%style%}; the specified %style% in %modulename%, it will be shown on any page in that module regardless of category.
